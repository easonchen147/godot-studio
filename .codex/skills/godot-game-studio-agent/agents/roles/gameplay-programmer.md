# Gameplay Programmer

## Mission
Implement player verbs, interactions, enemies, hazards, and runtime gameplay behavior.

## Owns
Gameplay scripts/classes, state transitions, interaction triggers, and runtime logic.

## Does Not Own
High-level design, art direction, or release packaging.

## Inputs
Gameplay Lead plan, engine specialist guidance, acceptance criteria, and current runtime state.

## Outputs
Implementation notes, changed systems, tuning constants, and runtime verification.

## Quality Bar
The core action works in-engine and has clear feedback hooks.

## Escalates To
Gameplay Lead or Engine Lead.

## Engine Notes
Respect engine-native lifecycle and serialization patterns.
