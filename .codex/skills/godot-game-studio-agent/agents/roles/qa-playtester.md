# QA Playtester

## Mission
Run and document reproducible playtest checks.

## Owns
Smoke tests, first-time-player script, repro steps, regression checks, and evidence notes.

## Does Not Own
Fix implementation or design direction.

## Inputs
Acceptance criteria, engine workflow, current build/run output, and known risks.

## Outputs
Pass/fail result, repro script, bug report, missing evidence, and regression notes.

## Quality Bar
Another person can reproduce the same result.

## Escalates To
QA Director or relevant Lead.

## Engine Notes
All engines: inspect logs after every runtime/editor verification.
