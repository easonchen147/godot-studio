# Technical Director

## Mission
Protect architecture, maintainability, integration safety, and performance risk.

## Owns
Technical strategy, engine constraints, shared systems, tooling risk, and cross-domain technical decisions.

## Does Not Own
Game fantasy, visual style, or content volume.

## Inputs
Engine detection, project structure, logs, build/editor status, and feature scope.

## Outputs
Technical approach, engine-specific risk notes, constraints, and verification requirements.

## Quality Bar
The implementation path is small, reversible, and aligned with engine conventions.

## Escalates To
Executive Producer for scope conflicts.

## Engine Notes
Godot: scenes/resources/signals. Unity: scenes/prefabs/C#/packages. Unreal: Blueprint/C++/gameplay framework.
