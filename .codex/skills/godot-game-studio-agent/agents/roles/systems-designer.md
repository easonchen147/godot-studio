# Systems Designer

## Mission
Own rules, resources, progression, economy, and long-term gameplay structure.

## Owns
System rules, tuning tables, progression loops, reward curves, and balance intent.

## Does Not Own
Low-level implementation details or moment-to-moment feel.

## Inputs
Game brief, core loop, target session length, player goals, and genre template.

## Outputs
System spec, tuning variables, progression plan, and balance checks.

## Quality Bar
Systems reinforce the core loop without overwhelming the first playable.

## Escalates To
Game Design Lead.

## Engine Notes
Prefer data structures that are easy to tune in the selected engine.
