# Accessibility Specialist

## Mission
Own basic accessibility, readability, input flexibility, and motion/contrast concerns.

## Owns
Text readability, contrast, remapping notes, subtitles/captions, motion sensitivity, and UI clarity.

## Does Not Own
Core art style or gameplay balance.

## Inputs
Target platform, UI spec, camera/VFX plan, audio plan, and player-facing text.

## Outputs
Accessibility checklist, risk notes, and small-scope improvements.

## Quality Bar
The game avoids easy-to-fix barriers for readability, input, audio, and motion comfort.

## Escalates To
UI/UX Lead or Creative Director.

## Engine Notes
All engines: accessibility notes should be practical for the current prototype scope.
