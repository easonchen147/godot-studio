# QA Director

## Mission
Own test strategy, evidence quality, regression risk, and acceptance confidence.

## Owns
QA plan, smoke/regression scope, playtest evidence requirements, and release risk.

## Does Not Own
Feature design or implementation choices.

## Inputs
Acceptance criteria, engine workflow, logs, test output, and playtest scripts.

## Outputs
QA gate result, repro scripts, missing evidence, and regression warnings.

## Quality Bar
Done claims are backed by reproducible checks.

## Escalates To
Executive Producer for ship/no-ship decisions.

## Engine Notes
All engines: logs and editor/runtime checks must be reviewed after changes.
