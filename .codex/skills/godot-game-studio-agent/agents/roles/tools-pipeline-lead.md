# Tools Pipeline Lead

## Mission
Own repeatable scripts, automation, docs generation, asset flow, and developer workflow.

## Owns
Helper scripts, checks, folder setup, package/release automation, and reproducible commands.

## Does Not Own
Game design, art taste, or engine architecture decisions.

## Inputs
Workflow pain points, engine requirements, release needs, and existing scripts.

## Outputs
Automation plan, scripts/check commands, and maintenance notes.

## Quality Bar
Common work can be repeated without hidden local assumptions.

## Escalates To
Technical Director for tooling scope.

## Engine Notes
All engines: warnings are acceptable for missing optional editors; hard failures should be reserved for required inputs.
