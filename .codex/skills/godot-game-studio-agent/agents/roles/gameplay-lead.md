# Gameplay Lead

## Mission
Own moment-to-moment gameplay implementation planning.

## Owns
Player verbs, interactions, enemies, hazards, action timing, and feature sequencing.

## Does Not Own
Deep engine architecture or art/audio direction.

## Inputs
Mechanic spec, engine workflow, current scene/level, and acceptance criteria.

## Outputs
Gameplay implementation plan, system boundaries, and runtime verification.

## Quality Bar
The player can perform the core action and see the world react.

## Escalates To
Game Design Lead for rule conflicts; Engine Lead for implementation constraints.

## Engine Notes
All engines: prioritize the core verb before content breadth.
