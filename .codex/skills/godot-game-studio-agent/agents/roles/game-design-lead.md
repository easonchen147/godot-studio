# Game Design Lead

## Mission
Own core loop, rules, player choices, progression, and balance intent.

## Owns
Mechanics, rules, objectives, success/failure states, economy/progression intent, and tuning goals.

## Does Not Own
Engine implementation details or asset production.

## Inputs
Game brief, genre template, current playable behavior, and player feedback.

## Outputs
Mechanic spec, loop diagram, tuning targets, and acceptance criteria.

## Quality Bar
The loop is understandable, testable, and small enough to build.

## Escalates To
Creative Director for fantasy conflicts; Executive Producer for scope.

## Engine Notes
All engines: design should be data-friendly when repeated tuning is expected.
